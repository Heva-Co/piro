# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities via GitHub issues.**

Email [devops@heva.co](mailto:devops@heva.co) with:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions (if known)

We aim to respond within **48 hours** and will keep you informed as we work toward a fix.

## Supported Versions

We release patches for the latest stable version only. If you are running an older version, we recommend upgrading before reporting.

## Disclosure Policy

We follow coordinated disclosure. Please give us reasonable time to address the issue before making it public. We will credit reporters in the release notes unless you prefer to remain anonymous.
